# 开发日志 — 2026-06-15

## 概述

为 exam-cram-agent 搭建 Koa HTTP 后端服务，以便前端（Vue 3）可以通过 API 调用 Agent 能力。

---

## 开发了什么

### 1. 后端 HTTP 服务（Phase 1）

基于 Koa 3 搭建了完整的 HTTP 服务端，目录结构：

```
src/server/
├── index.ts               # 入口：创建 Koa 实例、注册中间件、挂载路由
├── sse.ts                 # SSE 流工具函数
└── routes/
    ├── health.ts          # GET /api/health — 健康检查
    └── chat.ts            # POST /api/chat/stream — SSE 流式对话
```

### 2. RAG 初始化工具函数

将原来写在 `src/index.ts` 里的 `initRag()` 抽到独立的 `src/rag/init-rag.ts`，让 CLI 入口和 Koa 服务入口可以共用。

### 3. 入口改造

`src/index.ts` 新增 `--server` 参数模式，检测到该参数时启动 Koa 服务而不是 CLI。

---

## 为什么开发这些功能

- **前端需要后端**：原来只有 CLI 交互模式，用户没法通过浏览器使用
- **SSE 流式输出**：Agent 的流式回答需要实时推送到前端，不能等全部生成完再返回
- **RAG 知识库**：Agent 需要检索高数笔记来回答问题，后端启动时预加载知识库
- **健康检查端点**：方便前端和运维确认服务是否正常运行

---

## 解决了什么问题

| 问题 | 解决方案 |
|------|----------|
| Agent 只有 CLI 交互 | `--server` 参数启动 Koa HTTP 服务 |
| 前端需要实时看到 LLM 流式输出 | SSE（Server-Sent Events）通过 PassThrough 流推送 |
| 跨域请求被浏览器拦截 | 手动实现 CORS 中间件 |
| CLI 和 Koa 都要用 RAG 初始化 | 抽到 `init-rag.ts` 统一管理 |
| SSE 缺乏标准工具函数 | 封装 `createSSEStream` 和 `sendSSEEvent` 两个工具函数 |

## 踩坑记录

### 1. Koa 3 + CORS

**问题**：Koa 3 没有内置 CORS 支持，前端在 `localhost:5173` 请求 `localhost:3001` 被浏览器拦截。

**解决**：自己写一个简单中间件设置 `Access-Control-Allow-*` header，处理 OPTIONS 预检请求。不需要额外依赖。

### 2. SSE 的响应头

**问题**：一开始 `createSSEStream` 只设置了流，没设置 `Content-Type: text/event-stream`，前端可能无法正确识别 SSE。

**解决**：加上 `Content-Type`、`Cache-Control`、`Connection` 三个必要 header。

### 3. `process.exit(0)` 时机

**问题**：`src/index.ts` 的 `useKoa` 分支里，`await startServer()` 后面直接 `process.exit(0)`。但 `app.listen()` 是非阻塞的，调用后立即返回，紧接着 `process.exit(0)` 把进程杀了，curl 连不上。

**解决**：将 `server/index.ts` 的 `app.listen()` 包在 `new Promise` 里，在 listening 回调中 `resolve()`。同时将 `process.exit(0)` 改为 `await new Promise(() => {})` 保持事件循环。

### 4. CLI 用法检查误拦截

**问题**：`--server` 参数被 CLI 的 `if (!question && !useChat)` 用法检查拦截，打印帮助信息后 `process.exit(1)`。

**解决**：将 `if (useKoa)` 判断移到用法检查之前，让 Koa 模式提前 return。

### 5. RAG 加载时的 API Key

**问题**：`init-rag.ts` 从 `.env.local` 加载 API_KEY，但 Node.js 默认不读取 `.env` 文件，导致 `new OpenAI({apiKey: ""})` 构造函数抛出异常。

**解决**：在 `package.json` 的 `dev:server` 脚本中加 `--env-file=.env.local` 参数，同时在 `init-rag.ts` 里提前判断 API_KEY 是否为空，避免无效调用。

### 6. Koa 3 + @koa/bodyparser 兼容性

**问题**：Koa 3 是较新的版本，担心 `@koa/bodyparser` 是否兼容。实测 `@koa/bodyparser@6.1.0` 与 Koa 3.2.1 兼容。

### 7. OpenAI SDK 同步报错

**问题**：`new OpenAI({apiKey: ""})` 在构造函数中同步抛出异常（不是返回 rejected promise），需要在 try/catch 中捕获。

### 8. ESM 导入必须加 `.js` 后缀

**问题**：工程配置 `"type": "module"`，所有相对路径的 import 必须加 `.js` 后缀（如 `from "../rag/init-rag.js"`），否则运行时找不到模块。
