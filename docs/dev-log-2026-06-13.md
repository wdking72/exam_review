## 开发了什么

### 1. Agent 状态管理（Stage 5）— MemoryManager

```
src/core/memory.ts — Memory 管理：Sliding Window / Summarization
```

**两种策略**：

| 策略 | 行为 | 适用场景 |
|------|------|----------|
| Sliding Window | 保留最近 N 轮，超出的丢弃 | token 可控，但丢失历史 |
| Summarization | token 超阈值时，LLM 压缩旧轮为摘要 | 保留关键信息 |

**Token 估算**：本地估算，不调 API（中文 1.5 token/字，英文 0.25）。

### 2. 多轮对话 + 流式输出（Stage 6）

对 `agent-native.ts` 进行大改造：

| 方法 | 说明 |
|------|------|
| `agent.run()` | 单次运行（兼容旧调用） |
| `agent.chat()` | 多轮对话，保留 memory |
| `agent.init()` | 设置 system prompt，不触发对话 |
| `agent.streamChat()` | 流式输出 + 多轮对话 |

**流式 SSE 原理**：设置 `stream: true`，逐 chunk 接收 `delta.content`，通过回调立即推给调用方。

### 3. RAG 基础（Stage 1-3）

```
src/rag/
├── embedding.ts        # EmbeddingClient — 调用 Embedding API
├── vector-store.ts     # VectorStore — 内存向量库
├── chunker.ts          # 文档分块（Markdown 标题 + Overlap）
└── test-rag.ts         # 测试脚本
```

**三个核心模块**：

| 模块 | 功能 |
|------|------|
| EmbeddingClient | 调用 BAAI/bge-m3 模型，将文本转为向量 |
| VectorStore | 内存向量库，余弦相似度检索 Top-K |
| chunker.ts | 按 Markdown 标题切分 + Overlap |

### 4. 完整 RAG Pipeline（Stage 4）

```
src/rag/rag-engine.ts — RAGEngine 类
```

将分块 → 向量化 → 存储 → 检索串成一条流水线：

- `loadFromFile(path)`：读文件 → chunk → embed → 存入 VectorStore
- `search(question, topK)`：embed 问题 → 向量检索 → 格式化输出（来源 + 内容）

**Agent 集成**：`createTools(ragEngine?)` 根据是否传入 RAGEngine，注册真实 `search_knowledge_base` 或 mock 兜底。

## 为什么开发这些功能

- **Memory 管理**：多轮对话下 token 无限增长会撑爆 context window，必须裁剪
- **流式输出**：用户等不了 LLM 完整生成，需要逐 token 显示
- **RAG**：Agent 不能只靠 LLM 记忆回答问题，需要检索教材笔记
- **分块**：长文档必须切成适当大小才能检索

## 踩坑记录

### 1. finish_reason 被 null 覆盖

OpenAI 流式 API 末尾会发一个空 chunk（`finish_reason = null`），覆盖之前的值。加保护：`if (finishReason === null) 才赋值`。

### 2. addAssistant 重复调用

多个 tool call 时在循环里多次调用 `addAssistant`，memory 中出现重复 assistant 消息。修复：移到循环外统一加一次。

### 3. readline + pipe 兼容

`rl.question()` 在 pipe 关闭后抛 `ERR_USE_AFTER_CLOSE`。改用 `for await (const line of rl)` 模式。

### 4. SummarizerLLM 包装

Memory 压缩需要 LLM 接口，但 NativeToolAgent 用的是 OpenAI client。用 `SummarizerLLM` 类包装一层。

### 5. embedMany 返回值处理

`embedMany()` 返回 `number[][]`，不能 flat 拍平，每个元素对应一个输入文本的向量。

### 6. 余弦相似度除零保护

全零向量计算余弦相似度时除零，需要 `Math.max(len, 1e-10)` 保护。

### 7. Overlap 的边界

chunk 之间的 overlap 操作不当会导致信息重复或丢失。每个 chunk 头部追加前一个 chunk 的尾部内容。
